# Word processing: прототип пайплайна анализа вакансий

Этот блок содержит **экспериментальный (WIP) пайплайн** для обработки текстов вакансий HH и извлечения из них структуры.  
Код находится в **сыром виде**, активно дорабатывается и может меняться или ломаться без сохранения обратной совместимости.

- лемматизированный текст описания;
- найденные технологии/навыки по словарям;
- найденные категории (направления) и грейды;
- черновые метрики «релевантности» описания под дальнейшую аналитику.

Файлы и директории:

- `word_processing.py` — основной скрипт:
  - использует `natasha` (Segmenter, MorphVocab, NewsMorphTagger и др.) для лемматизации русского текста;
  - функция `lemmatization(text)` очищает сырой текст, приводит к леммам и возвращает список лемм;
  - функция `found_data_processing(text: list)`:
    - ищет совпадения в словарях `SKILL_MAP`, `CATEGORIES`, `GRADE_MAP`;
    - собирает найденные технологии по категориям;
    - считает простую оценку релевантности по количеству найденных навыков;
    - отбрасывает описания, где найдено слишком мало сигналов;
  - `main()` читает `vacancy_description.json` и прогоняет первые N вакансий через пайплайн, печатая разбор в консоль (формат для экспериментов, не для продакшена).
- `data.py` — словари и таксономии:
  - `GRADE_MAP` — ключевые слова для грейдов (Intern/Junior/Middle/Senior/Lead);
  - `SKILL_MAP` + `SKILL_ORDER` — карта технологий по категориям (языки, фреймворки, инфраструктура и т.д.);
  - `CATEGORIES` + `PRIORITY` — направления (AI/ML, QA, Cybersecurity, Analytics, Dev, Network, Education/HR, Support & Management) и приоритет их обработки.
- `test.py` — маленький пример, который проверяет, как словарь (`map`) матчится с одним конкретным текстом (ручной sanity‑check).
- `vacancy_description.json` — сырые описания вакансий (как пришли с HH API + HTML). Используется как вход для экспериментов.

Зачем это нужно (и почему пока сыро):

- протестировать **rule‑based** подход к разбору вакансий до интеграции с основным Streamlit‑дашбордом;
- собрать и отладить словари навыков/категорий/грейдов на реальных текстах;
- понять, достаточно ли качества rule‑подхода или потребуется подключать ML‑модель (см. `use model/`);
- экспериментировать с форматом данных (JSON/YAML), логикой фильтрации и метриками без гарантий стабильности.

Как использовать этот блок:

1. Подготовить `vacancy_description.json` (список объектов с полем `description`).
2. Запустить `word_processing.py` как отдельный скрипт:
   ```bash
   python word_processing.py
   ```
3. Смотреть лог в консоли: найденные навыки, категории, грейды, оценка релевантности.

Важно:

- это **экспериментальный R&D‑код в разработке**, не часть основного продакшн‑пайплайна Streamlit‑приложения;
- файлы с сырыми описаниями (`vacancy_description.json`, `vacancies_ready*.yaml`) могут быть тяжёлыми и завязаны на конкретные выгрузки — их не обязательно коммитить в публичный репозиторий;
- логика и словари здесь могут расходиться с тем, что используется на главной странице дашборда, и могут со временем быть перенесены/упрощены/удалены.
